Retour sur le besoin

Vous avez eu rendez-vous avec le biologiste qui avait passé commande du logiciel que vous venez de produire. Il est très satisfait de pouvoir enfin pouvoir séelctionner des fenêtre d’intérêt pour des facteurs de transcription dans les promoteurs des gènes qui l’intéressent.

Néanmoins il trouve que ce serait plus parlant avec une représentation graphique des résultats.

Après lui avoir demandé ce qu’il souhaiterait, il vous a préparé un petit schéma :

Chaque petit rectangle bleu reproduit une occurrence de la matrice JASPAR recherchée, chaque ligne le promoteur d’un des gènes passés en paramètre et le rectangle rouge montre une fenêtre sélectionnée.

Il indique que si en plus il pouvait y avoir une gradation de bleu suivant le score de la matricece serait super. Et si en plus on pouvait avoir une fenêtre surgissante indiquant le score et la position exacte, y compris pour la fenêtre d’intérêt, ce serait topissime.

Echange avec votre chef de projet

De retour avec votre chef de projet, celui-ci vous propose de prendre l’approche suivante.

  • on met de côté le ‘super’ et le ‘topissime’ dans un premier temps,
  • tant qu’à produire un affichage, autant le rendre indépendant du programme de calcul, on passera donc par un export des résulats qui seront ensuite rechargés pour l’affichage

En terme de technologie il vous indique de faire simple.

  • l’export sera réalisé via JSON (car en Python c’est facile : ne le faites pas « à la main » mais utilisez le module Python correspondant)
  • l’affichage dans un canevas Javascript (car en Javascript ce sera facile de charger le JSON)

Export des résultats

De manière générale, et c’est particulièrement vrai en bioinformatique, il est important que les résultats produits par un programme soit exploitables facilement. Pour cela les résultats doivent être produits dans un format standard, facilement lisible.

Pour cela faites produire à votre programme un fichier de sortie au format JSON (dont le nom sera spécifié par une option -o ou --output), qui contient à la fois les informations nécessaires pour exploiter les résultats, mais également les paramètres utilisés pour obtenir ces résultats (afin de garder la traçabilité de la façon dont ont été obtenus les résultats).

Vous décrirez ce format dans votre README.md.

On peut facilement exporter un dictionnaire Python au format JSON grâce au module JSON.

Affichage

Créer un fichier HTML/JavaScript avec un bouton permettant de charger un fichier JSON et de dessiner le contenu pour obtenir un affichage des résultats tel qu’attendu par le biologiste.
